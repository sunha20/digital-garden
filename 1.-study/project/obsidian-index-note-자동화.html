원하는 기능

(더 중요한 기능을 위로 정렬)

  1. index 자동 생성
  2. index 자동 업데이트
  3. graph view에 index note와 참조된 노트에 대한 link 표시 (markdown형식으로 노트 참조가 표시되어야함)
  4. 노트 정보(property) 표시
  5. 보기에 지저분 하지 않은 디자인 (ex. table)
  6. 새 노트 / 새 폴더 추가 버튼

사용할 plugin

  • folder note: 해당 폴더에 대한 정보를 담은 index note이기 때문에, 자동으로 folder note를 만들어주고, 접근하기 편하게 하는 해당 플러그인 사용.
  • 원하는 기능을 가진 플러그인 후보: templater, dataview, waypoint
도구/기능 자동 생성 자동 업데이트 link property format 자유도
templater O X O O 가장 놓음
dataview O O X O 중간 (LIST, TABLE, ONE LINE)
waypoint O O O X 가장 낮음 (LIST ONLY)
  • 완전히 만족하는 플러그인이 없어서 중요한 기능을 가장 많이 구현한 “waypoint”를 선택한 후 추가적인 수정 단계를 거침.

function convertDay(dt) {  
    switch (dt) {  
        case 0:  
            return "일";  
        case 1:  
            return "월";  
        case 2:  
            return "화";  
        case 3:  
            return "수";  
        case 4:  
            return "목";  
        case 5:  
            return "금";  
        case 6:  
            return "토";  
    }  
  
}  
  
function convertDate(dt) {  
    let dtStr = dt.toString()  
    if (dtStr.length === 1) {  
        return "0" + dtStr;  
    } else {  
        return dtStr;  
    }  
}  
  
function convertLink(v, k) {  
    if (typeof v === "string") {  
        if (v === "[[") return "";  
        if (v.startsWith("[[")) return v.replace("]]", `\\|${k}]]`);  
        if (v.startsWith("https://") || v.startsWith("http://")) return `[${k}](${v})`;  
    } else return v;  
  
}
  • 위의 수정본에서 생략된 부분으로 어떤 정보(property)를 어떤 형태(table)로 표시할지를 결정하는 부분
  • 표형식
    • 기본 개념
      • obsidian 에서 “|key1|key2| \n |---|---| \n |val1|val2|”의 형식으로 입력된 값이 자동으로 하단과 같은 표로 전환
      • 노트 작성자의 입력값을 받아오기 가장 쉬운 방법 중 하나는 노트의 property를 사용하는 것이다.
    • 신경 쓸 부분:
      • 어떠한 property가 표시될지는 폴더별로 다르기 때문에, index table이 생성될 folder note의 property가 통해 값을 입력받음.
      • default key(Title, Date)를 지정
      • 표형식의 경우 row마다 “|” 개수를 맞추어 주어야하기 때문에
        • 하위 노트가 index table에서 필요로하는 property가 빈값일 경우 그냥 “|”를 추가한다.
      • 폴더의 경우 받아올 만한 정보가 title정도이다
      • 날짜 정보는 유닉스 시간으로 받아오기 때문에 별도의 변환과정이 필요하다.
      • 웹링크의 경우 길이가 너무 긴경우가 있으니 임의로 변경해준다.
      • list 형식의 property는 띄어쓰기 없이 “,”로 구분되어 index table에 표시되기 때문에 “, ”로 변경해준다.
