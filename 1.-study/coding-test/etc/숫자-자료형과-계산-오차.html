숫자 자료형과 계산 오차

1072. 게임

참고: 위 수식이 틀린 이유는? (개발자 면접 타임) - YouTube

“컴퓨터는 무한한 숫자를 저장하지 못한다” 는 말은 쉽게 동의할 수 있는 명제이다.

그러나 이 의미를 정확히 숙지하지 않고 코드를 짜다간 단순히 큰 수를 표현하지 못하는 것으로부터, 계산 결과를 전혀 다르게 만드는 들 불안정한 코드가 완성되게 된다.

자료형

각각의 자료형은 고유한 메모리 크기를 가지고, 그 크기에 기반하여 해당 자료형이 표현할 수 있는 범위가 정해진다. 숫자를 표현하는 자료형은 경우 정수 자료형과 실수 자료형으로 나뉘고, 각각은 또 여러 개의 자료형으로 나뉜다.

정수 자료형

자바에서 정수를 표현하기 위한 자료형은 byte, short, int, long이 있다. 기본적으로 int 자료형을 사용한다.

type 메모리 크기 표현 범위 접미
byte 1 byte = 8 bit -128 ~ 128 → 개
short 2 byte -32,768 ~ 32,767 → 개
int 4 byte -2,147,483,648 ~ 2,147,483,647 → 개
long 8 byte -9,223,372,036,854,775,808 ~ 9,223,372,036,854,775,807 → 개 L

해당 범위 내의 값은 정확히 저장하지만, 표현범위를 벗어나는 값을 저장하는 경우 전혀 다른 값을 가지게 된다.

컴퓨터가 0과 1로 이루어진 이진수로 데이터를 표현한다고 하는 것을 유명하지만, 음수와 정수는 어떻게 표현할까?

실수 자료형

자바에서 정수를 표현하기 위한 자료형은 float와 double이 있다. 기본적으로 표현범위가 넓은 double을 사용한다.

type 메모리 표현 범위 유효자릿수 접미
float 4 byte (3.4 X 10^-38) ~ (3.4 X 10^38) 7 F
double 8 byte (1.7 X 10^-308) ~ (1.7 X 10^308) 16

이렇게 컴퓨터는 유한한 범위의 숫자만 저장이 가능하다.

문제는 이것으로 끝이 아니다. 위에서 살펴본 유한한 범위 내의 숫자라도 정밀하지 못하게 저장되는 경우가 있기 때문이다.
